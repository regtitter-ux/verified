# Vemoni API — интеграция для партнёров

Твой бот проводит верификацию **точно так же, как боты сети Vemoni**: показывает
рекламу, которую задаёт владелец, проверяет реальный заход участника на
сервер-спонсор и получает выплату за подтверждённые заходы. Всё, что влияет на
деньги (текст рекламы, ставка, кампании, лимиты, кран), контролирует владелец —
ты ничего из этого не настраиваешь.

Тебе нужны всего **две ручки**.

---

## Доступ

- **База:** `https://secure-gratitude-production.up.railway.app`
- **Авторизация:** заголовок в каждом запросе
  `Authorization: Bearer ТВОЙ_КЛЮЧ`
  (или `X-API-Key: ТВОЙ_КЛЮЧ`)
- Ключ выдаёт владелец. Он привязан к твоему аккаунту — все заходы и выплаты
  идут на твой баланс.

`SERVER_ID` ниже — это ID **твоего** сервера, где стоит верификация.

---

## 1. `GET /api/ad?serverId=SERVER_ID`

Что показать пользователю на твоём сервере прямо сейчас.

**Ответ:**
```json
{
  "adText": "# Зайдите на сервер: https://discord.gg/xxxx\n- После захода нажмите кнопку снова",
  "sponsor": { "guildId": "972405591140085791", "name": "GIFLAND СНГ", "invite": "https://discord.gg/xxxx" }
}
```

- `adText` есть → покажи этот текст юзеру, дождись «я зашёл», затем шаг 2.
- `adText: null` → рекламы сейчас нет → просто верифай (можно сразу шаг 2).

Реклама берётся по тому же правилу, что у всех ботов сети: персональная реклама
твоего сервера, если владелец её задал, иначе — глобальная. Меняется у владельца
— у тебя обновляется автоматически, хардкодить ничего не надо.

---

## 2. `POST /api/join-check`

Завершить верификацию. Тело:
```json
{ "userId": "ID_пользователя", "serverId": "SERVER_ID" }
```

`userId` твой бот берёт сам из Discord-события (`interaction.user.id`).

**Ответы:**

| Код | Тело | Что делать |
|-----|------|------------|
| `200` | `{ "joined": true, "credited": true }` | Заход подтверждён и засчитан → **пропускай юзера** |
| `200` | `{ "joined": true, "ad": false }` | Рекламы нет → верифай **без рекламы** → пропускай |
| `403` | `{ "joined": false }` | Не зашёл на сервер → попроси зайти и нажать снова |
| `503` | `{ "error": "..." }` | Временная ошибка → повтори запрос |

Сервер-спонсор определяется из **нашей** рекламы для твоего сервера, а не из
запроса — подменить нельзя.

## Полный цикл (пример на discord.js)

```js
const BASE = 'https://secure-gratitude-production.up.railway.app';
const KEY  = 'ТВОЙ_КЛЮЧ';
const auth = { Authorization: 'Bearer ' + KEY };

client.on('interactionCreate', async (i) => {
  if (!i.isButton() || i.customId !== 'verify') return;
  const userId = i.user.id;
  const serverId = i.guildId;

  // 1. Что показать
  const ad = await fetch(`${BASE}/api/ad?serverId=${serverId}`, { headers: auth }).then(r => r.json());
  if (ad.adText) {
    await i.reply({ content: ad.adText, ephemeral: true });
    // (покажи кнопку «Я зашёл» и вызови проверку по её нажатию)
  }

  // 2. Проверка захода (или сразу, если рекламы нет)
  const res = await fetch(`${BASE}/api/join-check`, {
    method: 'POST',
    headers: { ...auth, 'Content-Type': 'application/json' },
    body: JSON.stringify({ userId, serverId })
  });

  if (res.ok) {
    await giveRole(i.member);                       // пропускаем
  } else if (res.status === 403) {
    await i.followUp({ content: 'Сначала зайди на сервер, потом нажми снова.', ephemeral: true });
  } else {
    await i.followUp({ content: 'Попробуй ещё раз через пару секунд.', ephemeral: true });
  }
});
```

## Как это защищено (почему накрутить нельзя)

- Членство проверяется **напрямую через Discord** ботом сети, который сидит на
  сервере-спонсоре — фейковый заход не пройдёт.
- Проверяется **именно тот сервер**, что показан юзеру; членство в других
  спонсорах (например, с прошлых реклам) не засчитывается.
- Один участник по одному серверу оплачивается **один раз**.
- Если участник потом **уйдёт** с сервёра-спонсора — заход и выплата
  автоматически откатываются.
- Ставку, рекламу и кампании задаёт только владелец — ты их не меняешь.

---

## Деньги

- Начисление за заход идёт на твой баланс автоматически.
- Выплата: запрос формируется автоматически при достижении **$10**, либо
  мгновенная авто-выплата, если владелец её включил для тебя.
- Реквизиты выплаты: `PUT /api/requisites` с телом `{ "requisites": "USDT TRC20 <адрес>" }`.

## Прочие ручки (только чтение)

- `GET /api/balance` — баланс и реквизиты.
- `GET /api/stats` — твоя статистика верификаций (по серверам и окнам времени).
- `GET /api/withdrawals` — история выплат.
- `GET /` (на той же базе) — краткая справка по всем ручкам.
